# React前端项目说明

## ✅ 已完成的工作

### 1. 项目基础结构
- ✅ 使用Vite + React创建项目
- ✅ 配置Ant Design UI组件库
- ✅ 配置React Router路由
- ✅ 配置Axios进行API调用

### 2. API服务层（完全基于后端接口）
- ✅ `authService.js` - 认证相关API
- ✅ `organizationService.js` - 机构管理API
- ✅ `positionService.js` - 职位管理API
- ✅ `salaryItemService.js` - 薪酬项目管理API
- ✅ `employeeArchiveService.js` - 员工档案管理API
- ✅ `salaryStandardService.js` - 薪酬标准管理API
- ✅ `salaryIssuanceService.js` - 薪酬发放管理API

### 3. 核心功能
- ✅ 认证上下文（AuthContext）- 管理用户登录状态
- ✅ 路由配置 - 基于角色的路由保护
- ✅ 主布局 - 侧边栏菜单，根据角色动态显示
- ✅ 登录页面 - 完整的登录功能

### 4. 系统管理模块
- ✅ 一级机构管理（增删改查）
- ✅ 二级机构管理（增删改查）
- ✅ 三级机构管理（增删改查）
- ✅ 职位管理（增删改查）
- ✅ 薪酬项目管理（增删改查）

### 5. 员工档案管理模块
- ✅ 档案登记 - 完整的表单，支持机构级联选择
- ✅ 档案复核 - 待复核列表，支持通过和修改后通过
- ✅ 档案查询 - 多条件查询，支持分页
- ✅ 删除管理 - 删除和恢复功能

### 6. 薪酬标准管理模块
- ✅ 标准登记 - 支持添加多个薪酬项目
- ✅ 标准复核 - 支持通过和驳回

- ✅ 标准变更 - 查询标准后进行变更

### 7. 薪酬发放管理模块
- ✅ 发放登记 - 显示待登记机构列表
- ✅ 发放复核 - 支持通过和驳回，可修改明细
- ✅ 发放查询 - 多条件查询

## 🎯 技术特点

1. **完全基于后端API** - 所有接口调用都对应后端Controller
2. **统一响应处理** - 使用统一的ApiResponse格式
3. **Token认证** - 自动处理JWT Token
4. **权限控制** - 基于角色的菜单和路由保护
5. **错误处理** - 完善的错误提示和处理
6. **用户体验** - 使用Ant Design提供良好的UI体验

## 📦 安装和启动

```bash
# 1. 进入目录
cd ui-react

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

## 🔗 与后端对接

1. **后端服务** - 确保运行在 `http://localhost:8080`
2. **API地址** - 在 `src/services/api.js` 中配置
3. **CORS** - 后端已配置，允许前端跨域访问
4. **Token** - 登录后自动保存，请求时自动添加

## 📝 使用流程

1. 启动后端服务（IDEA中运行）
2. 启动前端服务（`npm run dev`）
3. 访问 `http://localhost:8000`
4. 使用测试账号登录
5. 根据角色使用相应功能
